= Data Infrastructure Insights AIアシスタント
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
*Data Infrastructure Insights AI Assistant* は、インフラストラクチャデータをより効率的に処理できるように設計されています。情報を分析し、洞察を発見し、運用上の課題を解決し、リソースに関する情報に基づいた意思決定を行うのに役立ちます。

[NOTE]
====
DII AI アシスタントは限定プレビューとしてご利用いただけます。Data Infrastructure Insights プレビュー機能link:/concept_preview_features.html["詳細情報"]。

AI アシスタントは、AWS にデプロイされた Data Infrastructure Insights テナントのみで利用できます。

====
image:ai_assistant_sample_question.png["AIアシスタントのサンプル質問"]



== インフラストラクチャの新しいインテリジェントパートナーをご紹介します

AI アシスタントは、データとのやり取りの方法を変革し、複雑なクエリを簡単な会話に変えます。隠れたパターンを即座に発見し、トラブルシューティングを加速し、データ主体の意思決定を自信を持って行うことができます。次のような質問に即座に回答が得られます：

* 許容できない遅延が継続した場合に警告するようにモニターを構成するにはどうすればよいですか？
* 環境内で使用率が最も高い上位のストレージプールは何ですか？
* 過去 1 時間に生成されたアラートの数はいくつですか？
* 収集されるログの最も一般的なタイプは何ですか？
* どのホストの CPU 使用率が最も高いですか？
* 環境内で監視できる資産タイプは何ですか？
* 現在アクティブな重大なアラートはありますか？
* ストレージシステムの全体的な健全性ステータスはどうですか？
* 最も多くのメモリを消費している仮想マシンはどれですか？
* システムにはいくつのモニターが設定されていますか？




== AIアシスタントを使用する前に

* AI アシスタントはあなたを支援するように設計されています。正確さを追求していますが、最良の結果を得るためには、出力を人間の監視下で検証することを強くお勧めします。
* あなたと同僚の安全とプライバシーのため、個人を特定できる情報（PII）を入力しないでください。
* データの安全を確保するために、機密情報や秘密情報は入力しないでください。
* link:https://www.netapp.com/pdf.html?item=/media/71584-netapp-data-processing-addendum.pdf["NetApp Data Processing補遺"]とlink:https://www.netapp.com/company/legal/privacy-policy/["NetAppプライバシー ポリシー"]をお読みください。




== AI アシスタントがあなたと連携します

DII AI アシスタントは、IT インフラストラクチャのデータに関する質問に答えるのに役立ちます。これを効果的に行うには、ユーザーのリクエストを理解するとともに、リクエストに応答するために必要な具体的なデータも理解する必要があります。

DII システムには、次のようなさまざまな種類のデータが含まれています：

* 資産テレメトリ（ホスト、ストレージ、VM など）
* データレイクのメトリクス（Kubernetes、NetAppストレージ、VMware）
* さまざまなシステムからのログ
* 異常を示す可能性のあるアラート


最適なエクスペリエンスを得るには、Assistant に次の具体的な情報を提供してください：

* 分析したいデータのタイプ（特定の資産、指標、システムなど）、
* 関心のある期間と
* 懸念している特定の問題の種類（パフォーマンスの問題、容量の問題、エラーなど）




== アシスタントへのアクセス

DII AI アシスタントを呼び出すには、画面の下隅にあるフクロウのアイコンを選択するだけです。「オーナー」ロールを持つユーザーは、テナント上の他のユーザーが AI アシスタントを利用する前に利用規約に同意する必要があります。
