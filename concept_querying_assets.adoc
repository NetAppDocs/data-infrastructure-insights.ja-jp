= アセットとメトリクスのクエリ
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
インフラストラクチャの物理資産と仮想資産をクエリすることで、パフォーマンスの監視、問題のトラブルシューティング、注釈などのカスタム基準に基づいたきめ細かな検索を実行できます。Data Infrastructure Insightsは、ストレージアレイやホストからアプリケーションや仮想マシンまで、多様な資産タイプを対象としたクエリに加え、Kubernetes、Docker、ONTAP Advanced Dataの統合メトリクスも提供し、包括的な可視性を実現します。

アセットに自動的にアノテーションを割り当てるアノテーション ルールを使用するには、クエリを全員に共有する必要があります。クエリの共有の詳細については、以下をご覧ください。

テナント上の物理または仮想インベントリ資産（および関連するメトリック）や、Kubernetes やONTAP Advanced Data などの統合によって提供されるメトリックをクエリできます。



== 在庫資産

すべてのインベントリ（インフラストラクチャとも呼ばれる）アセット タイプ（ストレージ、スイッチ、VM、アプリケーションなど）は、クエリ、ダッシュボード ウィジェット、カスタマイズされたアセット ランディング ページで使用できます。フィルター、式、表示に使用できるフィールドとカウンターは、アセット タイプによって異なります。



== 統合メトリクス

インベントリ資産とそれに関連するパフォーマンス メトリックのクエリに加えて、Kubernetes または Docker によって生成されたメトリックや、 ONTAP Advanced Metrics で提供されるメトリックなどの *統合データ* メトリックもクエリできます。

image:QueryPageFilter.png["統合クエリフィルターの例"]



== クエリの共有

クエリへのアクセスを制御するには、クエリを表示および編集できるユーザーを選択します。デフォルトでは、新しいクエリは非公開で、自分だけが閲覧できます。また、柔軟な権限レベル（Read-Only または Edit）を設定して、特定のユーザーまたは組織全体と共有することもできます。

クエリをすべてのユーザーと共有するか、編集権限または読み取り専用権限を持つ特定のユーザーと共有するかを選択できます。


NOTE: アカウント所有者権限を持つユーザーは、プライバシー設定に関係なく、すべてのクエリを表示できます。
